# Notes

Human-friendly operational notes for this project.

## Purpose
- Keep short, practical runbooks for repeatable testing and review.
- Version notes over time so process changes are traceable.

## Versioning convention
- Create one file per iteration: `vX-topic.md`.
- Keep old versions; do not overwrite history.
- If a process changes, add a new version file and reference the previous one.
